# 010: Многоуровневые подсказки (track: infra)

## Зависит от
000-conventions

## Что сделать
Расширить поддержку подсказок: поле `hints: list[str]` (1–3 уровня, от
наводящего вопроса до почти-решения) в TASK-словаре, с полной обратной
совместимостью со старым полем `hint: str`.

Файлы:
- `torch_judge/hints.py` — новая утилита `get_hints(task: dict) -> list[str]`:
  возвращает `task["hints"]` если есть, иначе `[task["hint"]]` если есть,
  иначе `[]`.
- Правки точек, где апстрим показывает hint (engine/web_engine/шаблоны):
  минимальные, через `get_hints()`. Если рендер подсказки в апстриме
  завязан на одну строку — показывать уровни по нарастающей (кнопка/вызов
  «ещё подсказка»); в ноутбуках уровни рендерятся как отдельные
  `<details><summary>Tip N</summary>...</details>` блоки.
- Генератор/обновлятор шаблонов (если в апстриме есть скрипт генерации
  templates — расширить его; если нет — добавить
  `scripts/render_hints.py`, вставляющий details-блоки в template по TASK).

## Поведение
- Задача апстрима с одним `hint` работает как раньше (визуально без
  изменений).
- Новая задача с `hints=[a,b,c]`: в template три сворачиваемых блока
  Tip 1..3; программно `get_hints()` возвращает все три по порядку.

## Приёмка
- `pytest tests/test_hints.py -q` — зелёный. Тесты: legacy-задача → 1
  подсказка; новая → 3 по порядку; ни того ни другого → [].
- `pytest -q` весь репозиторий — зелёный (апстрим не сломан).
- Демонстрационный template (любой один из будущего трека, можно
  фиктивный `templates/99_hints_demo.ipynb`, удалится позже) исполняется
  nbconvert'ом и содержит 3 details-блока.

## Не делать
Не менять формат хранения прогресса; не трогать существующие templates
апстрима.
