## Поведение
```python
def ddp_gradient_sync(
    rank_grads: list[dict[str, torch.Tensor]],  # градиенты N ранков,
) -> dict[str, torch.Tensor]                    # одинаковые наборы ключей
```
Вернуть усреднённые по ранкам градиенты (эквивалент allreduce-avg в DDP
после backward). Проверять согласованность: одинаковые ключи и shape у
всех ранков, иначе ValueError. None-градиенты (замороженный параметр:
None у ВСЕХ ранков) → None в результате; None только у части ранков →
ValueError.

В условии template — короткая теор. вставка: в настоящем DDP это
происходит бакетами с overlap'ом с backward; здесь — семантика результата.

## Тесты judge (минимум)
1. N=3, два параметра: результат allclose ручному среднему, atol=1e-6.
2. dtype/shape сохранены (bf16-градиенты остаются bf16; усреднение
   через float32 аккумулятор с касту обратно — допустимо и ожидаемо,
   проверить отсутствие потери точности на константных градиентах).
3. Замороженный параметр: None у всех → None в выходе.
4. Рассинхрон ключей между ранками → ValueError.
5. None только у одного ранка → ValueError.

## Подсказки (3 уровня)
1. «Что именно DDP синхронизирует и в какой момент шага?»
2. «Почему average, а не sum? Подумай про эквивалентность большому батчу».
3. Схема: stack по ранкам → mean(dim=0), внимание на None-кейсы.
